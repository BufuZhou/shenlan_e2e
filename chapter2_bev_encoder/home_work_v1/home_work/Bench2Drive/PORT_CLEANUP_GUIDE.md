# CARLA 端口清理机制说明

## 问题背景

之前的端口清理脚本只检查固定的两个端口（30002 和 50001），但项目中实际使用了多个不同的端口配置，导致清理不彻底。

## 改进方案

### 动态端口检测与清理

新的清理机制会：
1. **扫描所有可能的 CARLA 端口**
2. **只清理被占用的端口**
3. **多次尝试确保完全释放**
4. **最终验证所有端口状态**

## 清理流程

### 步骤 1：进程清理
```bash
bash tools/clean_carla.sh
```
- 杀死所有 `CarlaUE4-Linux` 进程
- 杀死包含 `carla-rpc-port` 的进程
- 杀死 `run_evaluation` 和 `leaderboard_evaluator` 进程

### 步骤 2：端口扫描
```bash
# 扫描所有 21 个端口
for port in 30001-30010 50000-50010; do
    pid=$(lsof -ti:$port)
    if [ ! -z "$pid" ]; then
        echo "端口 $port 被进程 $pid 占用"
    fi
done
```

### 步骤 3：针对性清理
只对**被占用的端口**进行清理：
```bash
for port in ${occupied_ports[@]}; do
    # 最多尝试 3 次
    for attempt in 1 2 3; do
        pid=$(lsof -ti:$port)
        if [ ! -z "$pid" ]; then
            kill -9 $pid
            sleep 1
        else
            break
        fi
    done
done
```

### 步骤 4：等待确认
```bash
sleep 10  # 等待 10 秒确保端口完全释放
```

### 步骤 5：最终验证
```bash
for port in 30001-30010 50000-50010; do
    pid=$(lsof -ti:$port)
    if [ ! -z "$pid" ]; then
        echo "错误：端口 $port 仍被占用"
        exit 1
    fi
done
```

## 优势对比

### 旧方案
❌ 只检查固定端口（30002, 50001）  
❌ 可能遗漏其他被占用的端口  
❌ 如果有残留进程使用其他端口，会导致冲突  

### 新方案
✅ 扫描所有可能的端口范围（30001-30010, 50000-50010）  
✅ 智能检测，只清理被占用的端口  
✅ 多次尝试确保完全释放  
✅ 最终验证所有端口状态  
✅ 清晰的错误提示，列出具体占用的端口

## 故障排查

### 问题 1：端口清理后仍被占用
**原因：** 进程无法被杀死（可能是僵尸进程或权限问题）  
**解决：**
```bash
# 手动查找并杀死进程
lsof -i :30002
kill -9 <PID>

# 如果还不行，检查是否有父进程
ps -ef | grep <PID>
```

### 问题 2：清理时间过长
**原因：** 等待 10 秒确认端口释放  
**解决：** 这是必要的等待时间，确保端口完全释放，不建议缩短

### 问题 3：误报端口占用
**原因：** 其他非 CARLA 程序使用了这些端口  
**解决：**
```bash
# 查看具体是什么进程在占用
lsof -i :30002
netstat -tlnp | grep 30002

# 如果不是 CARLA 进程，可以跳过或手动处理
```

## 扩展端口范围

如果需要支持更多端口，可以修改脚本中的数组：

```bash
# 增加 CARLA 端口范围
CARLA_PORTS=(30001 30002 ... 30020)  # 扩展到 30020

# 增加 TM 端口范围
TM_PORTS=(50000 50001 ... 50020)     # 扩展到 50020
```

## 最佳实践

1. **每次运行前自动清理** - 脚本已集成到 `start_eval_open_loop_wocontrol.sh` 开头
2. **异常退出后手动清理** - 使用 `tools/clean_carla.sh`
3. **定期检查端口状态** - 使用 `lsof -i :30000-30100`
4. **避免手动指定端口** - 让脚本自动管理端口
